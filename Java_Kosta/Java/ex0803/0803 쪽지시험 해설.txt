@@@1. Constructure 란?
 : 생성자라고 부르고 특별한 메소드이다. -> 기능, 선언해야한다 
 * 선언 방법 
   modifier 클래스이름(dataType 변수이름,....){
		
   }
*오버로딩이 가능하다 
  권장사항 : 기본생성자를 작성한 후에 생성자를 오버로딩하자 

* 자바의 모든 객체는 반드시 한개 이상의 생성자를 갖는다 
* 개발자가 생성자를 한개도 작성하지 않으면 자동으로 
  기본생성자가 삽입된다. 
-> 개발자가 생성자를 작성하면 기본생성자는 삽입되지 않는다
-> 기본생성자는 modifier 클래스이름(){} - 인수가 없는 생성자

*생성자는 직접 호출할 수 없다. 맘대로 호출 안된다.
 : 객체가 생성되는 시점에 자동 호출된다 -> 데이터 초기화를 위함 
   new 클래스이름(값, 값 ... );

* 생성자 구현부 첫번째 줄에서 this(값..); 을 이용해서
  같은 클래스 내 다른 생성자를 호출할 수 있다

*생성자를 사용하는 이유 : 객체가 생성되는 시점에 딱 한번 해야할 일이 
                                   있다면 메소드에 하는 것보다 
                                   생성자에서 하면 자동호출되어 편리함
                                   가장 많이 사용하는 생성자의 형태는
                                   객체가 생성되는 시점에 데이터 초기화 작업!
